# Contributing to Agent Governance Framework

Thank you for your interest in contributing to the Agent Governance Framework.

## How to Contribute

### Reporting Issues

- Use [GitHub Issues](https://github.com/governancelayer/agent-governance/issues) for bug reports and feature requests
- Use [GitHub Discussions](https://github.com/governancelayer/agent-governance/discussions) for questions, ideas, and open-ended feedback

### Submitting Changes

1. [Fork the repository](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/fork-a-repo)

2. Clone your fork:

   ```bash
   git clone https://github.com/YOUR-USERNAME/agent-governance.git
   cd agent-governance
   ```

3. Create a new branch:

   ```bash
   git checkout -b feature/your-feature-name
   ```

4. Make your changes

5. Push and submit a pull request to the main repository

### What We Value

- Small, focused changes
- Clear schema definitions
- Evidence-backed claims
- Neutral technical tone
- Concrete use cases over speculative architecture

### Specification Changes

Changes to governance primitive schemas in `spec/` should:

- Include a clear rationale for the change
- Provide example payloads demonstrating the change
- Not break existing valid examples without justification

### Documentation Changes

Documentation in `docs/` should:

- Be technically accurate and concise
- Include concrete examples where appropriate
- Follow the existing style and tone

## AI Contributions

If you are using any kind of AI assistance to contribute, this must be disclosed in the pull request or issue, along with the extent of AI involvement.

We welcome AI-assisted contributions, but human understanding and oversight of the changes is expected. See MCP's [contribution guidelines](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/CONTRIBUTING.md#ai-contributions) for a good model of this policy.
